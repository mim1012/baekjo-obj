# 백조오브제 — 실제 키값 전환 가이드 (클라이언트용)

> 대상: 백조 오브제 사업주(백보윤 대표님)
> 작성: 2026-07-16 / 담당 개발자: mim1012
>
> **이 문서의 목적**: 지금 사이트는 개발자 개인 계정으로 만든 "임시 키"와 "테스트 결제"로 돌아갑니다.
> 실서비스로 열려면 **사업자 명의로 발급받은 진짜 키**가 필요하고, 그 발급은 **사업주 본인 계정으로만** 가능합니다.
> 아래 4가지를 순서대로 진행해 주시면 됩니다.
>
> ⚠️ **발급받은 키는 절대 카카오톡·메일 본문에 그대로 붙여넣지 마세요.** 비밀번호와 같은 값입니다.
> 전달 방법은 §5를 참고해 주세요.

## 전체 그림 — 무엇을 왜 하나

| # | 할 일 | 왜 필요한가 | 대략 소요 |
|---|-------|-------------|-----------|
| 1 | 네이버 개발자센터 키 발급 | 사이트에서 "네이버로 로그인" 버튼이 동작 | 20분 + 검수 1~3영업일 |
| 2 | 카카오 개발자센터 키 발급 | "카카오로 로그인" 버튼이 동작 | 20분 + 비즈앱 심사 1~2영업일 |
| 3 | 도메인 발급(아임웹) | `baekjoobjet.com` 같은 **우리 주소**로 사이트 접속 | 30분 + DNS 반영 최대 24시간 |
| 4 | 토스페이먼츠 심사 신청 | 실제 카드 결제로 돈을 받을 수 있게 됨 | 서류 준비 1일 + 심사 3~7영업일 |

**권장 순서: 3번(도메인) → 1·2번(로그인) → 4번(결제).**
도메인이 먼저 정해져야 네이버·카카오에 등록할 주소가 확정되고, 토스 심사도 최종 도메인 기준으로 봅니다.
도메인 확정 전에 1·2번을 하면 나중에 주소를 다시 등록해야 하는 이중 작업이 생깁니다.

## 1. 네이버 개발자센터 — 실제 키값 발급

### 1-1. 가입

1. https://developers.naver.com 접속
2. 우측 상단 **로그인** → **사업주 네이버 아이디로 로그인**
   - ⚠️ **개인 아이디가 아니라, 앞으로 회사 대표로 쓸 아이디**로 하세요. 나중에 바꾸기 번거롭습니다.
   - 사업자 명의 네이버 아이디가 따로 없다면 지금 하나 만드는 걸 권합니다.
3. 개발자 이용약관 동의 → 휴대폰 본인인증

### 1-2. 애플리케이션 등록

1. 상단 메뉴 **Application** → **애플리케이션 등록**
2. 입력 항목:

| 항목 | 입력할 값 |
|------|-----------|
| 애플리케이션 이름 | `백조오브제` (로그인 동의 화면에 그대로 노출됩니다) |
| 사용 API | **네이버 로그인** 선택 |
| 제공 정보 선택 | **이메일 주소 · 이름 · 프로필 사진** (필수 체크) |
| 로그인 오픈 API 서비스 환경 | **PC 웹** 선택 |

3. **PC 웹**을 선택하면 아래 두 칸이 나타납니다. 정확히 입력하세요 (오타 한 글자면 로그인이 실패합니다):

| 칸 | 입력할 값 |
|----|-----------|
| 서비스 URL | `https://baekjo-obj.vercel.app` |
| 네이버아이디로로그인 Callback URL | `https://baekjo-obj.vercel.app/api/auth/callback/naver` |

> 📌 **도메인(§3)을 먼저 발급받으셨다면** 위 두 칸에 새 도메인도 **추가로** 등록해 주세요.
> 예: 서비스 URL에 `https://baekjoobjet.com`, Callback URL에 `https://baekjoobjet.com/api/auth/callback/naver`.
> Callback URL은 **여러 개 등록 가능**하니 기존 것을 지우지 말고 줄을 추가하시면 됩니다.

4. **등록하기** 클릭

### 1-3. 키 확인 (여기서 나오는 두 값이 "실제 키값")

등록 직후 **애플리케이션 정보** 화면에 표시됩니다.
(나중에 다시 볼 때: Application → 내 애플리케이션 → `백조오브제` → **개요** 탭)

| 화면에 표시되는 이름 | 개발자에게 전달할 이름 | 생김새 |
|---------------------|----------------------|--------|
| **Client ID** | `AUTH_NAVER_ID` | 영문+숫자 20자 내외 |
| **Client Secret** | `AUTH_NAVER_SECRET` | 영문+숫자 10자 내외 (**보기** 버튼을 눌러야 보입니다) |

### 1-4. 개발자를 멤버로 등록 (필수)

**멤버 관리** 탭 → 개발자 네이버 아이디를 **관리자**로 추가.
→ 이후 설정 변경 때마다 대표님께 로그인 요청드리지 않아도 됩니다.

> ⚠️ **검수 통과 전에는 여기 등록된 아이디만 네이버 로그인이 됩니다.**
> 즉 지금 단계에서 일반 고객은 네이버 로그인을 쓸 수 없습니다. 테스트하실 아이디도 여기 추가해 주세요.

### 1-5. 검수 신청 (오픈 직전에)

**API 설정** 탭 → **검수 요청**

- 필요한 것: **개인정보처리방침 페이지 URL** → 이미 사이트에 준비되어 있습니다: `https://baekjo-obj.vercel.app/privacy`
- 심사 기간: 보통 **1~3영업일**
- 통과하면 그때부터 **모든 고객**이 네이버 로그인 사용 가능

## 2. 카카오 개발자센터 — 실제 키값 발급

### 2-1. 가입 및 앱 생성

1. https://developers.kakao.com 접속
2. 우측 상단 **로그인** → **사업주 카카오 계정으로 로그인**
   - ⚠️ 여기도 **회사 대표로 쓸 카카오 계정**으로 하세요.
3. 상단 **내 애플리케이션** → **애플리케이션 추가하기**

| 항목 | 입력할 값 |
|------|-----------|
| 앱 이름 | `백조오브제` |
| 사업자명 | `백조 오브제` |
| 카테고리 | 쇼핑 |

4. **저장**

### 2-2. 키 확인 (여기서 나오는 첫 번째 값)

앱 생성 직후 **앱 키** 화면이 뜹니다.
(나중에 다시 볼 때: 내 애플리케이션 → `백조오브제` → **앱 설정 → 앱 키**)

| 화면에 표시되는 이름 | 개발자에게 전달할 이름 |
|---------------------|----------------------|
| **REST API 키** | `AUTH_KAKAO_ID` |

> ⚠️ 키가 4종류(네이티브 앱 키 / REST API 키 / JavaScript 키 / Admin 키) 나옵니다.
> **REST API 키 하나만** 필요합니다. 나머지는 보내지 마세요 (특히 Admin 키는 절대 공유 금지).

### 2-3. 카카오 로그인 켜기

1. 좌측 메뉴 **제품 설정** → **카카오 로그인**
2. **활성화 설정** → **ON**
3. 같은 화면 아래 **Redirect URI** → **등록** 클릭 → 아래 주소 입력:

```
https://baekjo-obj.vercel.app/api/auth/callback/kakao
```

> 📌 도메인(§3) 발급 후에는 `https://baekjoobjet.com/api/auth/callback/kakao` 도 **추가로** 등록.
> 여러 줄 등록이 가능합니다.

4. **저장**

### 2-4. Client Secret 발급 (두 번째 값)

1. **제품 설정 → 카카오 로그인 → 보안** 탭
2. **Client Secret** 항목의 **코드 생성** 클릭
3. 생성된 코드 아래 **활성화 상태**를 반드시 **사용함**으로 변경 → **저장**
   - ⚠️ 이 단계를 빼먹으면 로그인이 동작하지 않습니다. 가장 흔한 실수입니다.

| 화면에 표시되는 이름 | 개발자에게 전달할 이름 |
|---------------------|----------------------|
| **Client Secret 코드** | `AUTH_KAKAO_SECRET` |

### 2-5. 동의항목 설정

**제품 설정 → 카카오 로그인 → 동의항목**

| 항목 | 설정 |
|------|------|
| 닉네임 | **필수 동의** |
| 프로필 사진 | 선택 동의 |
| 카카오계정(이메일) | **필수 동의** ← **비즈 앱 전환 후에만 선택 가능** |

### 2-6. 비즈 앱 전환 (이메일 수집에 필수)

> **왜 필요한가**: 카카오는 "비즈 앱"으로 전환해야만 **고객 이메일을 받을 수 있습니다.**
> 전환 전에는 이메일 없는 회원이 생기고, 주문 확인 메일을 보낼 수 없습니다.
> 사이트 코드는 이 경우도 깨지지 않게 처리해 두었지만, **오픈 전에는 반드시 전환**하셔야 합니다.

1. 좌측 메뉴 **앱 설정** → **비즈니스**
2. **비즈 앱 전환** 클릭
3. 준비물: **사업자등록번호 `524-05-03658`** + 사업자등록증 사본
4. 심사: 보통 **1~2영업일**
5. 통과 후 → §2-5로 돌아가서 **이메일을 "필수 동의"로 변경**

### 2-7. 개발자를 팀원으로 초대 (필수)

**앱 설정 → 팀 관리** → 개발자 카카오 계정을 **관리자**로 초대.

## 3. 아임웹에서 도메인 발급

> **먼저 이해하실 것**: 도메인(`baekjoobjet.com` 같은 주소)과 사이트(홈페이지)는 **별개**입니다.
> 아임웹에서는 **도메인만 사서** 우리 사이트에 연결할 수 있습니다.
> 즉 **아임웹으로 홈페이지를 만드실 필요는 없고, 도메인 구매 기능만 쓰시는 것**입니다.

### 3-1. 도메인 이름 정하기 (제일 먼저 결정)

후보를 2~3개 준비해 주세요. 이미 남이 쓰고 있으면 못 삽니다.

- 추천 형태: `baekjoobjet.com` / `baekjo-objet.com` / `baekjoobjet.co.kr`
- `.com` = 국제 표준, 가장 무난 (연 2만원 내외)
- `.co.kr` = 한국 사업자 전용, 신뢰감 (연 2만원 내외, **사업자등록증 필요**)
- 🚫 하이픈·숫자가 많은 주소는 피하세요 (구두로 불러주기 어려움)

### 3-2. 아임웹에서 구매

1. https://imweb.me 접속 → 로그인 (없으면 가입)
2. 사이트 관리자 화면 → **설정** → **도메인** 메뉴
3. **도메인 구매(또는 도메인 검색)** → 원하는 이름 입력 → **사용 가능** 확인
4. 결제 (연 단위 결제, 자동 갱신 켜두시길 권합니다 — **만료되면 사이트가 통째로 접속 불가**)
5. 소유자 정보 입력: **사업자 정보 그대로** (상호 백조 오브제 / 대표 백보윤)

> 💡 **아임웹이 아니어도 됩니다.** 가비아·후이즈·Cloudflare 등 어디서 사도 동일하게 연결됩니다.
> 이미 아임웹 계정이 있으시면 아임웹이 편하고, 아니라면 **가비아**가 한국어 지원이 가장 무난합니다.
>
> ℹ️ 아임웹의 메뉴 이름은 개편에 따라 조금씩 달라질 수 있습니다. **설정 안에서 "도메인"** 이라는 단어만 찾으시면 됩니다.
> 못 찾으시면 화면을 캡처해 개발자에게 보내주세요.

### 3-3. 개발자에게 넘길 것 (이게 핵심)

도메인을 산 뒤, **DNS 설정**을 해야 우리 사이트에 연결됩니다. 둘 중 하나를 선택하시면 됩니다.

| 방법 | 하실 일 | 추천 |
|------|---------|------|
| **A. 개발자가 알려주는 값을 직접 입력** | DNS 레코드 2줄을 아임웹 **설정 → 도메인 → DNS 관리**에 입력 | ⭐ 소유권이 대표님께 그대로 남음 |
| **B. 계정 접근 위임** | 아임웹 계정에 개발자를 매니저로 초대 | 간편하지만 계정 공유 필요 |

**A 방법 진행 순서:**
1. 대표님: 도메인 구매 완료 → 개발자에게 **도메인 이름만** 알려주심 (예: `baekjoobjet.com`)
2. 개발자: 사이트가 올라가 있는 서비스(Vercel)에 도메인 등록 → **입력하실 값 2줄**을 알려드림
   - 대략 이런 형태입니다:
     ```
     타입: A      호스트: @      값: 76.76.21.21
     타입: CNAME  호스트: www    값: cname.vercel-dns.com
     ```
   - ⚠️ **실제 값은 반드시 개발자가 알려드리는 것을 쓰세요.** 위는 형태를 보여드리는 예시입니다.
3. 대표님: 아임웹 **설정 → 도메인 → DNS 관리(또는 레코드 관리)** → 위 2줄 입력 → 저장
4. 개발자: 연결 확인 (보통 10분~2시간, 최대 24시간 소요)
5. 완료되면 `https://baekjoobjet.com` 으로 사이트 접속됨 + 보안 자물쇠(SSL)는 자동 적용

> ⚠️ **아임웹으로 만든 홈페이지가 이미 그 도메인에 연결돼 있다면**, 위 설정을 바꾸는 순간
> 아임웹 홈페이지 대신 우리 사이트가 뜹니다. 기존 아임웹 사이트를 계속 쓰실 계획이라면
> **작업 전에 개발자와 먼저 상의**해 주세요 (서브도메인으로 분리 가능합니다).

### 3-4. 도메인이 정해지면 되돌아가서 할 일

- [ ] 네이버 개발자센터 → 서비스 URL·Callback URL에 새 도메인 **추가** (§1-2)
- [ ] 카카오 개발자센터 → Redirect URI에 새 도메인 **추가** (§2-3)
- [ ] 개발자: `APP_BASE_URL` 환경변수를 새 도메인으로 교체
- [ ] 토스페이먼츠 상점 정보의 사이트 주소도 새 도메인으로 (§4)

## 4. 토스페이먼츠 심사 신청

> **현재 상태**: 결제 기능은 **코드가 전부 완성**되어 있고 테스트 결제로 검증을 마쳤습니다.
> 지금 막혀 있는 건 **실제 카드 결제를 받을 계약(심사)** 하나입니다.
> 심사를 통과하면 개발자가 실제 키로 교체하는 데 10분이면 됩니다.

| 서류 | 비고 |
|------|------|
| 사업자등록증 사본 | 백조 오브제 / 524-05-03658 |
| 통신판매업 신고증 사본 | 2026-인천미추홀-0016 |
| 대표자 신분증 사본 | 백보윤 |
| **정산받을 통장 사본** | ⚠️ **반드시 사업자 명의 계좌** (개인 명의는 반려됩니다) |
| 대표자 휴대폰 | 본인인증용 |

### 4-2. 가입 및 신청

### 4-3. ⚠️ 심사에서 실제로 반려당하는 것들 — 미리 막아둔 것

카드사 심사는 **코드가 아니라 "사이트 화면"을 사람이 직접 보고** 판단합니다.
아래는 실제 반려 사유이고, 대부분 이미 처리해 두었습니다.

| # | 반려 사유 | 상태 |
|---|-----------|------|
| 1 | 판매 상품에 **가격이 표시되지 않음** | ✅ 해소 — 다만 현재 **99,000원은 임시값**입니다 (§4-4 참고) |
| 2 | 사업자 정보(상호·대표자·사업자번호·주소·연락처)가 사이트에 없음 | ✅ 해소 — 푸터·`/terms`에 실제 정보 표기 완료 |
| 3 | 이용약관·개인정보처리방침 페이지 없음 | ✅ 해소 — `/terms`·`/privacy` 게시 완료 |
| 4 | 교환·환불 정책이 명시되지 않음 | ✅ 해소 — 약관에 포함 |
| 5 | 상품 상세·주문·결제 화면이 실제로 동작하지 않음 | ✅ 해소 — 테스트 결제로 완주 검증 완료 |

### 4-4. 🔴 대표님이 반드시 하셔야 할 것 — **실제 판매가 확정**

**이것이 지금 심사를 막는 유일한 항목입니다.**

현재 진단 추천 상품 9개에 **99,000원이라는 임시 가격**이 들어가 있습니다.
심사관이 봤을 때 모든 상품이 똑같이 99,000원이면 "실제 판매 의사가 없는 사이트"로 판단해 반려될 수 있습니다.

**하실 일**: 상품별 실제 판매가를 정해서 알려주시거나, 관리자 화면에서 직접 입력해 주세요.

- 관리자 화면: `https://baekjo-obj.vercel.app/admin/products` → 상품 클릭 → 가격·재고 입력 → 저장
- 또는 **엑셀/메모로 "상품명 - 판매가 - 재고"** 만 정리해 주시면 개발자가 일괄 입력하겠습니다.

> 📌 참고로 현재 실서비스에 이미 대표님이 직접 입력하신 것으로 보이는 가격이 2건 있습니다
> (125,000원 1건 / 99,000원 1건). 이 값들도 최종 확정가인지 알려주시면 정리해 두겠습니다.

### 4-5. 심사 통과 후 (개발자 작업)

대표님은 하실 것 없고, 통과 알림만 전달해 주시면 됩니다.

1. 토스 개발자센터 → **API 키** → **라이브 키** 2종 확인
   - `TOSS_SECRET_KEY` (`live_sk_...`) — **절대 외부 노출 금지**
   - `NEXT_PUBLIC_TOSS_CLIENT_KEY` (`live_ck_...`)
2. 개발자가 서버 환경변수에 등록 → 테스트 키에서 교체
3. 웹훅 URL 등록 + 실카드 1건 소액 결제 → 취소까지 실측 검증
4. 그때부터 실제 결제 가능

## 5. 발급받은 키 전달 방법 (보안)

> 🔴 **키 = 비밀번호입니다.** 유출되면 남이 우리 이름으로 로그인 서비스를 쓰거나, 최악의 경우 결제를 조작할 수 있습니다.

**하지 마세요**
- ❌ 카카오톡·문자·이메일 본문에 그대로 붙여넣기
- ❌ 화면 캡처해서 단톡방에 공유
- ❌ 메모장 파일로 클라우드에 올리기

**이렇게 해주세요 (택1)**
1. ⭐ **가장 안전**: 키를 개발자에게 주지 말고, **개발자를 각 콘솔의 관리자로 초대**(§1-4, §2-7)해 주세요.
   그러면 개발자가 직접 키를 확인해서 서버에 넣습니다. **대표님이 키를 옮기실 필요 자체가 없습니다.**
2. 부득이하게 전달해야 한다면: 1회용 비밀 링크 서비스(onetimesecret 등)에 붙여넣고 링크만 전달
3. 최소한: 전화로 불러주시고, 메신저에는 남기지 않기

**Client Secret을 잃어버렸다면?** 다시 볼 수 없는 경우가 많습니다. 그냥 **새로 발급(재생성)** 하시고 개발자에게 알려주세요.

## 6. 최종 체크리스트

**대표님 (사업주)**
- [ ] 도메인 이름 후보 2~3개 결정
- [ ] 아임웹(또는 가비아)에서 도메인 구매 + 자동갱신 ON
- [ ] 개발자가 알려준 DNS 2줄 입력
- [ ] 네이버 개발자센터 가입 → 앱 등록 → **개발자를 멤버로 추가**
- [ ] 카카오 개발자센터 가입 → 앱 생성 → 로그인 ON → Client Secret 생성+**활성화** → **개발자를 팀원으로 초대**
- [ ] 카카오 **비즈 앱 전환** (사업자등록증)
- [ ] 토스페이먼츠 서류 4종 준비 (사업자등록증·통신판매업신고증·신분증·**사업자명의 통장사본**)
- [ ] 토스페이먼츠 가입 → 상점정보 입력 → 심사 신청
- [ ] 🔴 **상품별 실제 판매가 확정 → 전달 또는 관리자 화면 입력**

**개발자 (mim1012)**
- [ ] 도메인 등록 → DNS 값 안내
- [ ] 네이버·카카오 키 4종 + `AUTH_SECRET`을 운영 환경변수에 등록 + `NEXT_PUBLIC_SOCIAL_LOGIN=1`
- [ ] `APP_BASE_URL`을 최종 도메인으로 교체
- [ ] 네이버 검수 신청 (개인정보처리방침 URL 제출)
- [ ] 토스 라이브 키 교체 + 웹훅 등록 + 실카드 소액 결제·취소 실측
- [ ] 골든플로우 #2(구매)·#6(회원) 실화면 검증 + 증빙 스크린샷

## 7. 자주 나오는 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 카카오 로그인 시 `KOE006` 오류 | Redirect URI 미등록/오타 | §2-3의 주소를 **끝의 `/kakao`까지 정확히** 재확인 |
| 카카오 로그인이 그냥 실패 | Client Secret을 **생성만 하고 활성화 안 함** | §2-4의 "사용함" 설정 확인 |
| 네이버 로그인이 나만 되고 남은 안 됨 | 검수 전 상태 (정상) | §1-5 검수 신청 → 통과 대기 |
| 도메인 입력했는데 사이트가 안 뜸 | DNS 반영 대기 | 최대 24시간. 그 이후에도 안 되면 개발자에게 연락 |
| 결제 버튼이 "준비 중"으로 표시됨 | 토스 키 미등록 (정상, 안전장치) | 심사 통과 후 개발자가 키 등록하면 자동 활성화 |
