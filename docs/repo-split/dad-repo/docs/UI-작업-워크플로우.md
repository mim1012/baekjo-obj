# UI 작업 워크플로우 — 이 레포와 프로덕션의 관계

> 대상: 소유자(dad041566-hue)와 Antigravity 에이전트. 규칙 판정은 루트 [AGENTS.md](../AGENTS.md)가
> 정본이고, 이 문서는 "왜 그런 규칙인가"와 전체 흐름 그림이다.

## 전체 그림

```
[이 레포: BAGJO]                         [프로덕션: mim1012/baekjo-obj]
 UI 목업·디자인·레이아웃 (mock 데이터)      실서비스 (DB·API·결제)
        │
        │ ① dad: 화면 작업 → main 푸시 (build green)
        │ ② dad → mim: "올렸어" 한 마디
        ├──────── ③ mim: 표현 파일만 수확(diff) ────────▶  수확 PR 생성
        │                                                │ ④ mim: 실데이터 배선
        │                                                │ ⑤ CI: 빌드·시각회귀 게이트
        │ ⑥ dad: 프리뷰 링크 확인                          │
        │    "내가 올린 화면 그대로인가?" → 승인            │
        ◀──────── ⑦ mim: harvested/<날짜> 태그 ──────────┘ ⑧ 배포
```

## 왜 이렇게 하나 (한 레포에서 겪은 사고의 재발 방지)

한 레포에서 둘이 같은 파일을 만지던 시절, 병합 충돌을 풀다가 **dad의 리디자인이 4개 화면에서
통째로 유실**된 사고가 있었다(옛 디자인으로 조용히 회귀). 이 구조에서는:

- dad는 이 레포에서 **아무 눈치 없이** 화면만 만든다 — 브랜치 규칙, PR 라벨, CI 게이트 없음.
- 프로덕션의 표현 파일은 **mim의 수확 PR로만** 바뀐다 — 충돌 자체가 일어날 수 없다.
- 디자인이 그대로인지는 dad가 **화면(프리뷰)으로** 확인한다 — 코드 리뷰 불필요.

## 소유자(사람)가 기억할 것 — 딱 3개

1. **푸시 전 `npm run build`** 한 번. (깨진 채 올라오면 mim이 어디까지가 의도인지 알 수 없다)
2. 화면에 넣을 글자·목록·가격은 **컴포넌트 밖(props나 파일 상단 상수, `src/data/*`)에** 두라고
   에이전트에게 시키기. (mim이 실데이터로 갈아끼우는 자리가 그곳이다)
3. `src/lib`·`src/app/api`·`src/types`·`supabase` 폴더는 **없는 셈 치기.** 에이전트가 그쪽을
   고치려 들면 중단시키기. (여기 고쳐봐야 프로덕션에 반영 안 되고, mim 배선과 꼬이기만 한다)

## 에이전트에게 자주 시킬 일과 표준 지시문

| 하고 싶은 것 | 지시 예 |
|--------------|---------|
| 화면 리디자인 | "홈 히어로 섹션을 ~~ 느낌으로 리디자인해줘. AGENTS.md 구역 규칙 준수" |
| 새 섹션 + 새 데이터 | "브랜드 상세에 수상내역 섹션 추가. 필요한 필드는 src/data/brands.ts mock에 임의로 추가하고 커밋에 [wiring] 표시" |
| 신규 페이지 | "이벤트 페이지 신설. page.tsx 생성은 [wiring] 커밋으로 분리" |
| 올리기 전 점검 | "git status에서 🔴 구역 변경이 섞였는지 확인하고, npm run build 돌린 뒤 커밋·푸시" |

## 하지 말 것 (사고 이력에서 나온 것들)

- `harvested/*` 태그 삭제·이동 금지 — mim의 수확 기준점이다. 지우면 다음 수확에서 변경분을 놓친다.
- `src/types/index.ts` 수정 금지 — "타입 에러가 나서 고쳤다"가 가장 흔한 꼬임 경로다.
  타입 에러가 나면 mock(`src/data/*`) 쪽을 타입에 맞추거나, 그래도 안 되면 mim에게 필드 추가를 요청.
- `package.json`에 라이브러리 추가 금지 — 프로덕션에 없는 의존성은 수확 시 화면이 깨진다.
  새 라이브러리가 꼭 필요하면 먼저 mim과 합의(양쪽에 같이 추가).
- 이 레포의 `.github/workflows`·`tests`를 고치거나 지우지 않기 — 스냅샷 잔재라 무해하다. 무시가 정답.

## 문의 창구

배선·데이터·계정·배포 관련은 전부 mim1012. 이 레포에서 해결하려고 하지 말 것.
