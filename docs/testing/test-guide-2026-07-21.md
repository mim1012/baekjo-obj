# 2026-07-21 테스트 가이드

런칭 직전 수동/자동 QA에서 빠지면 안 되는 항목을 날짜별로 누적한다. 이 문서는 실행 결과가 아니라
검증할 시나리오와 합격 기준이다.

## 1. 비회원 주문 차단

목표: 상품 탐색과 장바구니 담기는 비회원도 가능하되, 주문 생성은 회원 세션 없이는 절대 DB에 저장되지
않아야 한다.

### 구현 대상

- `POST /api/orders`는 회원 전용이어야 한다. 세션이 없으면 `401`과 `login-required` 에러를 반환하고,
  `orders.member_id = null` 신규 주문을 만들면 안 된다.
- `/cart`의 `주문하기` 진입은 비회원이면 로그인으로 보내고, 회원이면 `/checkout`으로 이동해야 한다.
- `/checkout` 직접 접근도 비회원이면 로그인으로 보내야 한다. 장바구니는 로컬에 남아 로그인 후 이어서
  주문할 수 있어야 한다.
- `/login?redirect=/checkout`로 들어온 회원은 로그인 성공 후 `/checkout`으로 복귀해야 한다.
- 기존 결제/주문 테스트 중 인증 없이 `/api/orders`를 호출하는 케이스는 회원 로그인 또는 인증 쿠키를
  포함하도록 바꿔야 한다.
- 약관/문서의 "비회원 주문", "게스트 주문 허용" 문구는 회원 주문 전용 정책과 맞게 수정해야 한다.

### 합격 기준

1. 비로그인 브라우저에서 `/shop` 상품 탐색과 장바구니 담기는 가능하다.
2. 비로그인 브라우저에서 `/cart`의 `주문하기`를 누르면 로그인 화면으로 이동한다.
3. 비로그인 브라우저에서 `/checkout`을 직접 열어도 로그인 화면으로 이동한다.
4. 비로그인 상태로 `POST /api/orders`를 직접 호출하면 `401`이 반환되고, 상품 재고와 `orders` 행 수가
   변하지 않는다.
5. 회원 로그인 후 같은 장바구니로 `/checkout`에 복귀해 무통장입금 주문을 생성할 수 있다.
6. 생성된 주문은 `orders.member_id`가 로그인 회원 id로 저장되고, 마이페이지 주문 목록에 표시된다.
7. 관리자 주문 목록/상세에서도 같은 주문이 보이며, 입금확인/배송상태 변경이 기존처럼 작동한다.
8. 과거에 이미 존재하는 `member_id = null` 주문은 삭제하지 않는다. 관리자 이력으로 남기되, 새 비회원
   주문만 차단한다.

### 권장 자동화

- 소스 계약 테스트: 비로그인 `POST /api/orders`가 `401`을 반환하는 단위/라우트 테스트.
- 실구동 테스트: 비회원 `/cart` → 로그인 유도 → 회원 로그인 → `/checkout` 복귀 → 무통장 주문 생성 →
  마이페이지/관리자 반영 확인.
- 결제 라우트 테스트: 기존 인증 없는 주문 생성 helper를 회원 세션 기반으로 전환.

## 2. 상품 상세페이지 이미지/텍스트 등록 UX

목표: 관리자가 상품 상세 본문을 네이버 스마트스토어식으로 `이미지 → 텍스트 → 이미지` 순서로 쉽게
구성하고, 저장 직후 공개 상품 상세에서 같은 순서로 확인할 수 있어야 한다.

### 현재 구조 확인

- `/admin/products/new`와 `/admin/products/[id]`의 기본 상품 폼에는 `대표 이미지`와 `추가 이미지 갤러리`가
  있다. 이 `추가 이미지 갤러리`는 상세 본문 편집기가 아니라 상품 상세 상단 갤러리에 노출되는 이미지
  배열이다.
- 실제 상세 본문은 `/admin/products/[id]/editor`의 `ProductDetailEditor`가 `detailBlocks`로 저장한다.
  여기에서 `텍스트 추가`, `이미지 추가`, 드래그 정렬, 위/아래 이동, 실시간 미리보기가 가능하다.
- 즉, 기능상으로는 텍스트를 이미지 사이에 넣을 수 있지만, 신규 상품 등록창에서는 이 경로가 명확하지
  않아 관리자가 "이미지 1, 이미지 2, 이미지 3이 순서대로만 올라가는가?"라고 오해하기 쉽다.

### 합격 기준

1. 기존 상품에서 `상세페이지 편집`을 열 수 있다.
2. `이미지 추가`로 업로드한 이미지는 `detailBlocks`의 image 블록으로 저장된다.
3. `텍스트 추가`로 입력한 문구는 image 블록 사이에 배치할 수 있다.
4. 위/아래 버튼 또는 드래그로 `이미지 → 텍스트 → 이미지` 순서를 바꿀 수 있다.
5. 저장 후 `/shop/[id]`의 상세 본문이 에디터 미리보기와 같은 순서로 렌더된다.
6. 빈 이미지 블록/빈 텍스트 블록은 저장 시 제외되며, 기존 정상 블록을 지우지 않는다.
7. 업로드 실패, 허용되지 않은 이미지 URL, 너무 큰 상세 본문은 사용자에게 명확한 오류로 보여야 한다.

### UX 개선 후보

- 상품 수정 화면에서 바로 `상세페이지 본문 편집하기` CTA를 보여준다.
- 기본 상품 폼의 `추가 이미지 갤러리` 설명을 "상단 갤러리용"으로 더 명확히 바꾸고, 상세 본문은 별도
  에디터에서 만든다는 안내를 붙인다.
- 상세 에디터의 블록 라벨을 `Image Block`/`Text Block` 대신 `이미지 1`, `텍스트 1`처럼 한국어와 순번으로
  표시한다.
- 각 블록 하단에 `여기에 텍스트 추가`, `여기에 이미지 추가` 버튼을 둬서 맨 아래 추가 후 재정렬하지
  않아도 되게 한다.
- 여러 이미지를 한 번에 선택하면 이미지 블록을 순서대로 여러 개 생성하고, 각 이미지 사이에 텍스트
  블록을 삽입할 수 있게 한다.
- 저장 전 미리보기에서 공개 상세와 같은 폭/여백으로 보여 실제 노출 결과와 차이를 줄인다.
