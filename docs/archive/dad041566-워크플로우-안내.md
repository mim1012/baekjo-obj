## 3. 왜 이렇게 하나요? — 오늘 있었던 일

오늘 `codex/baekjo-site-launch`의 8개 커밋을 main에 머지하는 과정에서 **lint 에러 9건 + 빌드(타입) 에러 2건**이
발견되어 제가 수동으로 고쳤습니다. 대표 사례:

- `admin/partners` mock 데이터가 `status: 'active'`(영문)로 작성됨 → 계약 타입은
  `'운영중' | '상담중' | ...`(한글 리터럴)이라 **빌드 실패**. `type: 'hotel'`도 타입에 없는 값이었습니다.
- `admin/survey`의 recommendation 객체에 `categorySlug`, `needInsuranceAnalysis`, `recommendKit` 필드 누락.

**PR을 거쳤다면 GitHub CI가 push 몇 분 안에 자동으로 잡아줬을 문제들**입니다. 브랜치에 직접 push만 하면
CI가 돌지 않아서, 문제가 머지 시점에야 터집니다. PR 흐름은 통제가 아니라 서로의 시간을 지키는 안전망입니다.
