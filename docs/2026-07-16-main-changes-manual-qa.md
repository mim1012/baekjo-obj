## 3. 사용자가 직접 수동 검증해야 할 항목

### P0 — 토스 심사/결제 관련 최우선

#### 3-1. 카드결제 위젯 노출 확인
- 경로: `/shop` → 상품 상세 → 장바구니 또는 바로구매 → `/checkout`
- 확인:
  - `카드결제 준비중` 문구가 없어야 함.
  - `카드결제` 선택 가능해야 함.
  - 토스 결제수단 위젯과 약관 동의 위젯이 보여야 함.
  - 실제 결제창 진입이 가능해야 함.
- 주의:
  - 현재 키는 테스트키이므로 실제 과금은 되지 않음.
  - 심사 제출 전에는 토스/카드사 담당자에게 테스트 결제 가능 경로를 안내해야 함.

#### 3-2. 주문 금액 일치 확인
- 경로:
  1. `/shop`에서 상품 가격 확인
  2. `/cart` 장바구니 금액 확인
  3. `/checkout` 최종 결제금액 확인
  4. 주문 생성 후 `/admin/orders` 금액 확인
  5. `/admin/orders/[id]` 상세 금액 확인
- 확인:
  - 상품가 + 배송비 = checkout 최종 결제금액.
  - checkout 금액 = 관리자 주문 목록 금액.
  - 관리자 목록 금액 = 관리자 상세 결제 정보 금액.
  - 5,900원 상품을 주문하면 관리자에 5,900원 기준으로 보여야 함.
  - 68,600원 상품을 주문하면 관리자에 68,600원 기준으로 보여야 함.

#### 3-3. 배송/환불/사업자정보 심사 표면 확인
- 경로:
  - `/terms`
  - `/privacy`
  - `/refund-policy`
  - `/shop/p1` 같은 상품 상세
  - `/checkout`
  - 페이지 footer
- 확인:
  - 사업자 정보가 푸터에 보이는지.
  - 이용약관/개인정보처리방침/배송·교환·환불 안내 링크가 footer에 보이는지.
  - 사업자정보확인 링크가 새 탭으로 열리는지.
  - 상품 상세에 배송비/출고 일정/교환·반품 정책이 보이는지.
  - checkout에서 주문 전 확인 체크박스가 필수인지.
