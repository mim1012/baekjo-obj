# Baekjo Platform Completion Research

## Overview

기존 백조오브제 Next.js 16 프로젝트를 유지하면서 첨부 기획서와 대조해 누락된 화면, 데이터 필드, localStorage 흐름, 관리자 콘솔을 보완한다.

## Existing Strengths

- 사용자 핵심 라우트 22개와 공통 카드/내비게이션이 이미 존재한다.
- 고민 8개, 브랜드 5개, 상품 12개, 후기 6개, 공지 6개 mock data가 존재한다.
- 장바구니, 주문 저장, 보험 신청 저장, 찜 기능이 localStorage 기반으로 연결돼 있다.
- 최근 적용한 아이보리·스톤·세이지 디자인 시스템과 반응형 공통 레이아웃을 유지할 수 있다.

## Confirmed Gaps

- `/signup`과 관리자 7개 필수 라우트가 없다.
- 홈에 검증 브랜드, 라이프스타일, 독립 보험/전문가 섹션이 부족하고 고민 카테고리가 4개만 노출된다.
- 쇼핑 필터/정렬이 기획서보다 적고 상품 카드에 고민 태그가 없다.
- 상품 상세 Q&A와 추천상품 탭, 브랜드 후기 영역이 부족하다.
- 주문완료가 실제 저장 주문을 표시하지 않는다.
- 보험 신청 동의/관심 보장 필드와 관리자 메모 흐름이 없다.
- 로그인 저장, 회원가입, 마이페이지 전체 메뉴가 부족하다.
- 관리자 공통 사이드바, 검색/상단바, 리소스별 운영 화면이 부족하다.

## Recommended Approach

기존 Server Component 페이지는 유지하고 localStorage가 필요한 폼·구매·관리 화면만 Client Component로 둔다. 기존 타입은 하위 호환 필드를 남겨 과거 저장 데이터도 읽을 수 있게 확장한다. 관리자 화면은 공통 운영 테이블 컴포넌트로 일관되게 구성한다.

## Risks

- 기존 localStorage 데이터는 새 필드가 없을 수 있으므로 모든 읽기 화면에서 fallback을 사용한다.
- 관리자 UI는 실제 API가 없으므로 변경 가능한 상태만 localStorage로 처리하고 나머지는 mock 운영 UI로 명확히 표현한다.
- Next.js 16의 `params`와 `searchParams` Promise 규칙을 유지한다.
