# docs/evidence

**용도**: §8-4 납품 증빙 보관처. 마일스톤마다 골든플로우(AGENTS.md §7) 수동 점검 결과를
스크린샷·녹화로 여기 남긴다 — 배포/작업물이 아니라 **"이 시점에 이게 됐다"는 증거**다.
분쟁(환불 등) 방어 근거로 §8-4가 요구하는 자료가 이 폴더다.

## 하위 폴더 규칙

마일스톤(또는 배포일)당 폴더 하나, `YYYY-MM-DD-<마일스톤명>/` 형식으로 만든다.

```
docs/evidence/
  2026-07-20-phase1-delivery/
    01-diagnosis-flow.png
    02-shop-checkout.mp4
    README.md   ← 그 마일스톤에서 확인한 골든플로우 번호·통과 기준·확인자·날짜
```

- 폴더 안 `README.md`에는 최소한 다음을 적는다: 확인 대상 골든플로우(§7 번호), 통과 기준,
  확인 환경(프리뷰 URL 또는 prod), 확인자, 날짜.
- 파일명은 골든플로우 번호나 화면명이 드러나게 순번을 붙인다(`01-`, `02-` …).
- 이 폴더는 사실 기록이다 — 지나간 마일스톤 폴더의 내용을 수정하지 않는다(추가만).
